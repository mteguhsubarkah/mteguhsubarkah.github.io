Project Overview

A Dynamic Positioning System (DPS) is an advanced marine control system that automatically maintains a vessel's position and heading using its own thrusters, without the need for anchors or mooring lines. The system continuously receives data from GPS, compass, gyroscopes, anemometers, and other environmental sensors, processes it through sophisticated control algorithms in a DP computer, and generates real-time commands for each thruster.

As the Control System Engineer, I was responsible for the complete development of both the DP Software and DP Monitoring Software — from mathematical modeling and control algorithm design to software implementation and full-scale vessel deployment. The project involved computational fluid dynamics (CFD) for vessel modeling, implementation of multiple advanced control algorithms (PD, LQR, Sliding Mode, MPC), and real-time embedded system programming.

The system was successfully deployed on multiple submarine cable laying vessels, enabling precise station-keeping during critical cable installation operations where positioning accuracy is paramount.

Control System Architecture

Multi-layered control approach for robust autonomous positioning

1

State Estimation

Nonlinear observer filters noisy GPS and compass signals, estimates environmental bias, and performs wave-motion filtering to provide clean state feedback for the controller.

Noise elimination Wave filtering Bias estimation
2

Feedback Control

Multiple control algorithms implemented and tested. Linear controllers (PD, LQR) for basic operation, nonlinear controllers (Sliding Mode, MPC) for robust performance under varying sea conditions.

PD Proportional-Derivative
LQR Linear Quadratic Regulator
SMC Sliding Mode Control
MPC Model Predictive Control
3

Wind Feedforward

Proactive wind compensation before the vessel deviates. Receives real-time anemometer data, calculates wind force and moment, and generates feedforward signals to counteract disturbances.

Proactive compensation Force/moment calculation Disturbance rejection
4

Control Allocation

Converts control signals to individual thruster commands (RPM + azimuth angle) using pseudo-inverse transformation matrices. Thrust-to-RPM relationship calibrated through bollard pull tests.

Pseudo-inverse allocation Bollard pull calibration RPM + Azimuth commands
5

Thruster Control

Azimuth thrusters with 360° thrust vectoring. Hydraulic drive (~550 HP diesel) for propeller speed via swash plate axial pump. Induction motor with VFD-based position control for azimuth angle.

360° azimuth Hydraulic RPM control VFD azimuth positioning

Challenges & Solutions

Engineering obstacles overcome in building a production DP system

Challenge

Noisy Sensor Signals

GPS and compass signals contain significant noise that causes erratic control signals and potential actuator damage if fed directly to controllers.

Solution

Implemented nonlinear observer for state estimation with wave-motion filtering and environmental bias estimation, providing clean signals for feedback control.

Clean state feedback without actuator damage

Challenge

Vessel Nonlinearity & Environmental Disturbances

Vessel dynamics are inherently nonlinear, and sea conditions (wind, waves, current) constantly change, making linear controllers insufficient.

Solution

Implemented multiple nonlinear controllers (Sliding Mode, MPC) with wind feedforward compensation. CFD-based vessel modeling provided accurate dynamics for controller design.

Robust positioning under varying sea conditions

Challenge

Thruster-to-RPM Calibration

Converting desired thrust force to actual propeller RPM commands requires accurate characterization of the thruster's nonlinear thrust curve.

Solution

Conducted bollard pull tests at various RPMs to map thrust-to-RPM relationship. Used regression analysis to create accurate lookup tables for real-time control allocation.

Accurate thrust delivery from RPM commands

Challenge

Azimuth Motor Oscillation

Using contactors for azimuth motor control caused oscillation and potential damage to the actuator and relay systems.

Solution

Designed custom VFD (Variable Frequency Drive) with PID position control for the induction motor azimuth system, eliminating oscillation and providing smooth, precise angular positioning.

Smooth 360° azimuth control without oscillation

Key Takeaways

Lessons from building a production dynamic positioning system

01

Simulation Before Deployment Saves Everything

MATLAB/Simulink simulation (MIL-S) was invaluable for tuning control gains and identifying edge cases before putting the system on a real vessel. Field debugging is expensive and dangerous.

02

State Estimation is as Important as Control

A perfect controller with noisy input produces terrible results. The nonlinear observer for noise filtering was crucial — without it, the thrusters would oscillate and damage mechanical components.

03

Theory vs Reality Always Has a Gap

MPC should theoretically outperform other controllers, but gain tuning in real-world conditions with model uncertainty remained challenging. Practical robustness matters more than theoretical optimality.

04

Hardware-Software Integration is an Art

Bridging the gap between control algorithms and physical hardware (hydraulic pumps, VFDs, induction motors) requires understanding both domains deeply. Pure software or pure hardware expertise isn't enough.
